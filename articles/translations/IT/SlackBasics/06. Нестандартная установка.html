6. Нестандартная установка


Порой вам может понадобиться установить Slackware Linux нестандартным способом, например, чтобы получить лучшее понимание о том, как работает система GNU/Linux, или для подготовки сценария автоматической установки. Эта глава наметит шаги, которые необходимы для установки Slackware Linux вручную. Пример установочного скрипта также приведён в части 6.5 «Сценарий автоматической установки»

6.1. Разметка жёсткого диска

Если вы уже устанавливали ОС обычным способом, то вы не должны столкнуться с проблемами во время разметки жёсткого диска. Вы можете использовать команды fdisk и cfdisk. Если вы автоматизируете установку Slackware Linux, будет полезным знать, как вы можете передавать команды для обработки fdisk. Например:

user@darkstar:~

  • # fdisk /dev/hda << EOF
  • n
  • p
  • 1
  • +10000M
  • n
  • p
  • 2
  • +512M
  • t
  • 2
  • 82
  • w
  • EOF

Эти команды создают основной раздел Linux размером 10 000 Mb и swap-раздел размером 512 Mb. Вы можете сохранить команды fdisk в различных дисковых профилях и использовать их в различных установках (выбирая по размеру или иному критерию). Например:

user@darkstar:~

  • # cat /usr/share/fdisk-profiles/smalldisk | fdisk

6.2. Инициализация и подключение файловых систем

После создания разделов на жёстком диске, вы можете создать файловые системы и подключить их для использования. На системах с небольшим объёмом памяти, вы должны сперва подключить swap. В дальнейших примерах мы будем использовать созданную выше дисковую конфигурацию. Для подключения swap-раздела выполните:

user@darkstar:~

  • # mkswap /dev/hda2
  • # swapon /dev/hda2

Смысл этих команд совершенно ясен. mkswap создаёт swap, а swapon подключает его. Вы должны использовать mkswap лишь единожды, но swapon будет выполняться при каждой загрузке системы. Это может делаться автоматически, если вы добавите swap-раздел в файл /etc/fstab, что мы сделаем чуть позже.

Теперь нужно провести инициализацию разделов. Это можно сделать с помощью команды mkfs. Вы можете указать требуемый тип файловой системы в параметре -t. mkfs автоматически выполнит команду mkfs.<тип_файловой_системы> в зависимости от указанного параметра. Знайте, что доступные файловые системы зависят от типа установки загруженного ядра ОС. Если вы загрузились с ядра bare.i, то вам будут доступны типы ФС ext2, ext3 и reiserfs.

Чтобы инициализировать файловую систему ext3 и подключить её для использования, выполните следующие команды:

user@darkstar:~

  • # mkfs -t ext3 /dev/hda1
  • # mount /dev/hda1 /mnt

Если вы создали отдельные разделы для различных директорий файловой системы, например, /home, вы можете инициализировать их и подключить в соответствующую директорию с помощью следующих команд:

user@darkstar:~

  • # mkfs -t ext3 /dev/hda2
  • # mkdir /mnt/home
  • # mount /dev/hda2 /mnt/home

Наконец, вы должны примонтировать носитель с дистрибутивом. Это просто сделать, если вы используете CD-ROM. Допустим, что CD-ROM соответствует файл устройства /dev/hdc, тогда вы можете подключить его так:

user@darkstar:~

  • # mount /dev/hdc /var/log/mount

Использование NFS в качестве источника для установки ОС требует дополнительных действий. Для начала вы должны загрузить сетевой диск. При доступности физического носителя, вы можете сделать это с помощью команды network. Вы также можете загрузить этот диск с другого носителя, например, с USB-накопителя. Допустим, у вас есть USB-накопитель, подключённый к директории /var/log/mount, тогда вы можете загрузить сетевой диск с помощью команды network /var/log/mount/network.dsk. После загрузки диска, вы должны настроить сетевые интерфейсы. Если NFS-сервер находится в той же сети, вам требуется только назначить интерфейсу IP-адрес. Например, для использования адреса 192.168.2.201, нужно выполнить следующую команду:

user@darkstar:~

  • # ifconfig eth0 192.168.2.201

Затем вы можете загрузить portmapper, который необходим для корректной работы NFS-клиента:

user@darkstar:~

  • # /sbin/rpc.portmap

Если portmapper запустился успешно, вы можете подключить NFS-диск. Но сначала убедитесь, что вы отключили диски, ранее примонтированные к директории /var/log/mount. Допустим, что вы хотите подключить ресурс 192.168.2.1:/home/pub/slackware-current, тогда вы должны выполнить следующую команду:

user@darkstar:~

  • # mount -r -t nfs -o nolock 192.168.2.1:/home/pub/slackware-current /var/log/mount

Если не возникло ошибок во время монтирования раздела, тогда он будет доступен в директории /var/log/mount

6.3. Установка пакетов

Сейчас всё готово к установке пакетов с установочного носителя. Вы можете использовать команду installpkg, доступную в системе, для установки пакетов Slackware Linux. Для установки пакетов в файловую систему, подключённую к директории /mnt, добавьте параметр -root. Следующая команда установит все пакеты с медианосителя.

user@darkstar:~

  • # installpkg -root /mnt /var/log/mount/slackware/*/*.tgz

Если вы создали файл тегов, описывающий пакеты, предназначенные для установки, тогда вы можете его использовать (создание файлов тегов описано в части 17.4. «Управление пакетами»). Допустим, что файлы тегов для каждого дискового набора хранятся в директории /usr/share/tagfiles/small-desktop, тогда вы можете установить все указанные пакеты следующим образом:

for p in a ap d e f k kde kdei l n t tcl x xap y; do
   installpkg -infobox -root /mnt \
     -tagfile /usr/share/tagfiles/small-desktop/$p/tagfile \
     /var/log/mount/slackware/$p/*.tgz
done

6.4. Настройка после установки

Следующий раздел описывает минимально необходимые настройки, которые нужно сделать в запущенной системе.

6.4.1. fstab

Один из необходимых шагов конфигурации системы – создание файла fstab, в котором описываются дисковые разделы и устройства, подключаемые при загрузке системы. Формат файла /etc/fstab описан в разделе 8.8.4. Как минимум, вы должны добавить запись для корневой файловой системы, для псевдо-систем /proc и devpts и для раздела swap.

Для разделов, созданных ранее в этой главе, вы должны создать такой файл /etc/fstab:

user@darkstar:~

  • # cat > /mnt/etc/fstab << EOF
  • /dev/hda2 swap swap defaults 0 0
  • /dev/hda1 / ext3 defaults 1 1
  • devpts /dev/pts devpts gid=5,mode=620 0 0
  • proc /proc proc defaults 0 0
  • EOF
6.4.2. LILO

Для того, чтобы сделать систему загружаемой, вы должны настроить и установить загрузчик Linux (LILO, Linux Loader). Настройка LILO описана в разделе 19.1. Загрузчик. Здесь же мы просто покажем пример конфигурации LILO, которая может быть использована с разделами, созданными в этой главе.

user@darkstar:~

  • # cat > /mnt/etc/lilo.conf << EOF
  • boot = /dev/hda
  • vga = normal
  • timeout = 50
  • image = /boot/vmlinuz
  • root = /dev/hda1
  • label = Slackware
  • read-only
  • EOF

После настройки LILO вы можете установить его в раздел /mnt. В этом случае LILO будет использовать /etc/lilo.conf с целевого раздела:

user@darkstar:~

  • # lilo -r /mnt
6.4.3. Сеть

Настройка сети в Slackware Linux описана в Главе 22. В этом разделе мы опишем один пример настройки узла сети с получением IP-адреса с помощью DHCP.

Информация об известных компьютерных сетях хранится в файле /etc/networks. Так как мы будем получать сетевую информацию с помощью DHCP, мы укажем только локальную сеть 127.0.0.1.

user@darkstar:~

  • # cat > /mnt/etc/networks << EOF
  • loopback 127.0.0.0
  • localnet 127.0.0.0
  • EOF

Не смотря то, что DHCP назначит и имя хоста, мы всё же укажем его в файле:

user@darkstar:~

  • # cat > /mnt/etc/HOSTNAME << EOF
  • sugaree.example.net
  • EOF

Когда имя компьютера настроено, оно и узел localhost должны быть добавлены в файл /etc/hosts для разрешения имён:

user@darkstar:~

  • # cat > /mnt/etc/hosts << EOF
  • 127.0.0.1 localhost
  • 127.0.0.1 sugaree.example.net sugaree
  • EOF

Мы не создали файл /etc/resolv.conf, т.к он будет создан автоматически с помощью DHCP-клиента dhcpcd. Для завершения настройки мы должны настроить интерфейс в файле /etc/rc.d/tc.inet1.conf:

user@darkstar:~

  • # cat > /mnt/etc/rc.d/rc.inet1.conf << EOF
  • IPADDR[0]=""
  • NETMASK[0]=""
  • USE_DHCP[0]="yes"
  • DHCP_HOSTNAME[0]=""
  • EOF

Вы можете захотеть сделать резервную копию файла rc.inet1.conf, т.к. он содержит много полезных комментариев. Или же вы можете использовать sed для изменения конкретных параметров:

user@darkstar:~

  • # sed -i 's/USE_DHCP\[0\]=""/USE_DHCP[0]="yes"/' \
  • /mnt/etc/rc.d/rc.inet1.conf
6.4.4. Тонкая настройка сценариев инициализации

В зависимости от назначения установленной системы, нужно определить, какие именно сценарии инициализации будут выполняться в процессе загрузки. Состав доступных для загрузки сервисов зависит от установленных пакетов. Вы можете вывести список имеющихся сценариев с помощью ls:

user@darkstar:~

  • # ls -l /mnt/etc/rc.d/rc.*

Если на файле скрипта стоит бит исполнимости, то он будет выполнен, в противном случае сценарий будет пропущен. Очевидно, что вы должны оставить включенными стандартные необходимые скрипты, включающие сценарии, соответствующие уровням исполнения. Вы можете установить на файл бит исполнимости так:

user@darkstar:~

  • # chmod +x /mnt/etc/rc.d/rc.scriptname

Или удалить его:

user@darkstar:~

  • # chmod -x /mnt/etc/rc.d/rc.scriptname
6.4.5. Настройка кэша динамических библиотек

GNU/Linux использует кэш для загрузки динамических библиотек. Кроме того, множество программ зависят от основных версий библиотек (например, от /usr/lib/libgtk-x11-2.0.so, а не от /usr/lib/libgtk-x11-2.0.so.0.600.8). Кэш и символьные ссылки на библиотеки могут быть обновлены с помощью запуска ldconfig:

user@darkstar:~

  • # chroot /mnt /sbin/ldconfig

Возможно, вам не знакома команда chroot; это команда, которая исполняет другие команды с корневой директорией, отличной от корня текущей файловой иерархии. В этом примере корневой директорией будет /mnt, и в ней chroot запустит файл /sbin/ldconfig. После того, как программа завершит свою работу, система вернётся в командную оболочку, которая использует оригинальный корневой каталог. Для использования chroot с другими командами, нужно, как в примере, сперва запустить ldconfig, потому что без инициализации привязок динамического компоновщика программы могут не запуститься из-за неразрешимых библиотечных зависимостей.

6.4.6. Установка пароля root

Сейчас, когда настроен кэш динамических библиотек, вы можете выполнять команды в установленной системе. Мы воспользуемся этим и зададим пароль суперпользователя. Команда passwd может быть использована для установки пароля существующего пользователя (пользователь root по умолчанию настроен в файле /etc/passwd). Мы снова воспользуемся командой chroot, чтобы запустить passwd root на требуемом разделе:

user@darkstar:~

  • # chroot /mnt /usr/bin/passwd root
6.4.7. Настройка часового пояса

На UNIX-системах очень важно установить корректный часовой пояс, так как он используется различными частями операционной системы. Например, временная зона используется NTP для синхронизации системного времени, или различными сетевыми программами для вычисления разницы во времени между клиентом и сервером. На Slackware Linux временная зона устанавливается созданием ссыки /etc/localtime на файл зоны. Вы можете найти временную зону вашего региона в директории /mnt/usr/share/zoneinfo. Большая часть временных зон может быть обнаружена в поддиректории соответствующего региона. Например, для использования временной зоны Амстердама, вы можете выполнить следующие команды:

user@darkstar:~

  • # cd /mnt
  • # rm -rf etc/localtime
  • # ln -sf /usr/share/zoneinfo/Europe/Amsterdam etc/localtime

После установки часового пояса, программы всё равно не будут знать, является ли время, установленное на аппаратных часах, местного или же по стандарту Всемирного координированного времени (UTC, Coordinated Universal Time). Если вы используете другую операционную систему на этом же компьютере, то лучше не использовать UTC и установить локальное время. На UNIX-подобных системах традиционно используется системное время UTC. Вы можете указать тип системных часов путём создания файла /etc/hardwareclock и указания в нём слова localtime в случае использования местного времени или UTC в ином случае. Например, для использования UTC, вы можете создать файл /etc/hardwareclock следующим образом:

user@darkstar:~

  • # echo "UTC" > /mnt/etc/hardwareclock
6.4.8. Создание кэша шрифтов X11

Если вы планируете использовать X11, вы должны инициализировать кэш для шрифтов TrueType и Type1. Это можно сделать с помощью команды fc-cache:

user@darkstar:~

  • # chroot /mnt /usr/bin/fc-cache

6.5. Сценарий автоматической установки

Можно объединить шаги установки в одном сценарии, чтобы устанавливать ОС в автоматическом режиме. Это идеально для создания одного сервера установки, с помощью которого можно развёртывать множество Linux-клиентов. Эта часть содержит пример сценария, который был написан Вильямом Парком. Очень просто добавить установочный скрипт на дистрибутивный носитель Slackware Linux, особенно если вы используете установочные CD или загружаетесь с USB-накопителя.

Система установки хранится в одном сжатом файле, доступном на загрузочном диске как isolinux/initrd.img. Вы можете скопировать этот образ на жёсткий диск и распаковать его с помощью gunzip:

user@darkstar:~

  • # mv initrd.img initrd.img.gz
  • # gunzip initrd.img.gz

После распаковки образа, вы сможете смонтировать файл в качестве дискова, используя устройство loopback:

user@darkstar:~

  • # mount -o loop initrd.img /mnt/hd

Сейчас вы можете добавить скрипт в файл initrd, поместив его в структуру директорий, подключённую к точке монтирования /mnt/hd. После необходимых изменений, вы можете отключить файловую систему и сжать её:

user@darkstar:~

  • # umount /mnt/hd
  • # gzip initd.img
  • # mv initrd.img.gz initrd.img

Далее вы можете скопировать новый файл initrd.img на установочный диск и проверить сценарий.

	    /usr/sbin/gpm -k
	    sleep 1
	    /usr/sbin/gpm -m /dev/mouse -t ps2
	    ;;
	start)
	    echo "Starting gpm..."
	    /usr/sbin/gpm -m /dev/mouse -t ps2
	    ;;
	*)
	    echo "Usage $0 {start|stop|restart}"
	    ;;
esac
EOF
chmod +x etc/rc.d/rc.gpm
