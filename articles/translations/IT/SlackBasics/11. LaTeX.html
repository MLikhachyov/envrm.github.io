11. LaTeX


11.1 Введение

LaTeX – это система набора текста, которая может быть использована для создания высококачественных статей, книг, писем и других публикаций. LaTeX основан на TeX, низкоуровневом типографическом языке, созданном Дональдом Е. Кнутом. LaTeX не работает как текстовые процессоры WYSIWYG, системы подготовки документов, которые используют большинство людей. С LaTeX вы можете не заботиться о форматировании документа, а сосредоточиться на его написании.

Файлы LaTeX – это обычные текстовые файлы, содержащие макросы LaTeX. LaTeX форматирует документ в соответствии с используемыми вами макросами. В самом начале использования LaTeX может быть немного неудобным. Но по мере изучения, вы откроете, что использование LaTeX определенные преимущества. Назовем некоторые из них:

  • Документы, созданные в LaTeX выглядят очень профессионально.
  • Вы не должны заботиться о компоновке документа. Вы просто добавляете структуру в ваш документ и LaTeX берет на себя заботы о форматировании.
  • LaTeX-файлы – это простой текст, и они могут быть изменены с помощью стандартных утилит UNIX, таких как vi, sed или awk.
  • LaTeX предоставляет обеспечивает хорошую поддержку набора текста: математические формулы, сноски и Postscript-изображения.

LaTeX очень обширен. Эта часть только коснется поверхности LaTeX. Но этого вполне хватит для создания простых документов.


11.2. Подготовка простого документа в LaTeX

11.2.1. Минимальная структура документа

Каждый документ LaTeX имеет базовую минимальную структуру, описывающую документ. Взгляните на пример:

\documentclass[10pt,a4paper]{article}

\title{The history of symmetric ciphers}
\author{John Doe}

\begin{document}

This is a basic document.

\end{document}

Вы можете увидеть основные синтаксические структуры команд LaTeX. Команда начинается с обратной косой черты, за которой следует имя команды. Каждый мкрос имеет обязательный аргумент, который располагается в фигурных скобках, и опциональные аргументы, располагающиеся в квадратных скобках.

  • Первая комманда в каждом документе – это documentclass. Данная команда определяет, с каким видом документа LaTeX работает. Тип документа – обязательный параметр. Также вы можете указать необязательные параметры, такие как размер шрифта и размер бумаги. В примере размер шрифта переопределен со стандартных 12p на 10p, а размером бумаги указан лист формата A4.
  • После класса документа вы можете добавить метаинформацию о, например, заголовок документа. В примере заголовком является «The history of symmetric ciphers».
  • Команда author определяет автора книги.
  • Команда \begin отмечает начало окружения. Существует множество различных окружений, которые определяют соглашения по набору текста. В примере мы начинаем окружение document. Это очень простое окружение, LaTeX интерпретирует все содержимое данного окружения как текст документа.
  • Содержимое документа может быть расположено внутри меток document.
  • Все окружения должны быть закрыты. Окружение document является последним закрытым окружением, т.к. оно отмечает конец текста документа.

Как сообщила команда LaTeX, файл DVI создан после запуска команды latex. Вы можете просмотреть этот файл с помощью графического просмотрщика DVI-файлов, xdvi:

user@darkstar:~

  • $ xdvi crypto.dvi

Этот файл не совсем пригоден для печати (хотя с помощью xdvi можно распечатать документ). Идеальный формат для печати – Postscript. Вы можете получить Postscript-файл из файла DVI с помощью одной простой команды:

user@darkstar:~

  • $ dvips -o crypto.ps crypto.dvi

Параметр -o определяет выходной файл для Postscript-документа. Если этот параметр не указан, вывод будет направлен команде lpr, которая поставит документ в очередь на печать.

PDF – еще один популярный формат электронных документов. Его можно легко получить из Postscript:

user@darkstar:~

  • $ ps2pdf crypto.ps

Итоговый выходной файл будет назван как и файл-источник, но с расширением .pdf.

11.2.3. Разделы

Вы уже знаете, как создать простой документ LaTeX. Самое время чуть больше структурировать текст. Структура добавляется с помощью использования разделов, подразделов и подподразделов. Структурные элементы создаются с помощью соответствующих команд: \section, \subsection и \subsubsection. Обязательный параметр раздела - заголовок. По умолчанию разделы автоматически нумеруются и добавляются в оглавление. Если добавить звездочку (*) после команды создания раздела, например, \section*{title}, данный раздел не пронумеруется и не будет добавлен в оглавление. Следующий пример показывает, как вы можете использовать разделы:

\documentclass[10pt,a4paper]{article}

\title{The history of symmetric ciphers}
\author{John Doe}

\begin{document}

\section{Pre-war ciphers}

To be done.

\section{Modern ciphers}

\subsection*{Rijndael}

Rijndael is a modern block cipher that was designed
by Joan Daemen and Vincent Rijmen. 

In the year 2000 the US National Institute of
Standards and Technologies selected Rijndael as the
winner in the contest for becoming the Advanced
Encryption Standard, the successor of DES.

\end{document}

Приведенный пример очень прост, но он показывает, как LaTeX обрабатывает переводы строк и пустые строки. Переводы строк игнорируются LaTeX'ом, так что текст не прерывается. Пустые линии начинают новый абзац. Все абзацы начинаются с отступа слева.

11.2.4. Начертание шрифтов

Вы можете захотеть работать с различными начертаниями шрифтов. LaTeX имеет несколько команд, которые используются для изменения вида текущего шрифта. Наиболее используемые команды - это \emp для выделения текста и \textbf. Посмотрите Таблицу 11.2 «Начертание шрифтов в LaTeX» для получения списка стилей шрифтов. Выделение и полужирное начертание текста демонстрируется в следующем абзаце:

Working with font styles is easy.
\emp{This text is emphasized} and
\textbf{this text is bold}.
