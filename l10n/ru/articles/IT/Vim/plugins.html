Расширения Vim


Vim многогранен и удивителен. Кривая обучения работе в этом редакторе – притча во языцех, но если заставить себя выучить основные мнемокоманды, то набор текста, будь это программный код, заметка или документ LaTeX, становится лёгким как никогда. Говорят, что курсор следует за взглядом вимера, и это почти правда. Vim – не «операционная система со встроенным текстовым редактором», но он очень расширяем, хоть и язык сценариев – VimL – не лучший пример языков программирования. Каждый день на официальном сайте http://www.vim.org появляются новые расширения. Я долгое время работаю в Vim, и из многих сотен испробованных дополнений отобрал лучшие (естественно, на моё усмотрение и сугубо для моих целей).

Тонкую настройку Vim и всех перечисленных расширений можно найти в репозитории на github, здесь же я только перечислю то, чем пользуюсь сам.



1. Pathogen блегчает установку расширений. Не использую Vundle и прочие подобные расширения, т.к. не вижу смысла в функции автоматического обновления и управления расширениями из Vim (вместо файлового менеджера).

2. Vim-Airline – «продвинутая» строка состояния. Использую старую версию, т.к. мне не нужна интеграция с другими расширениями.

3. Cmdline-increment – инкремент/декремент чисел в командной строке с помощью стандартных комбинаций клавиш: Ctrl + a и Ctrl + x. Особенно полезно для записи макросов.

4. Speeddating – тоже инкремент, но это расширение работает уже с датами и временем в разных форматах.

5. Comment Toggle – комментирование строк. Лучший плагин из всех опробованных. Поддерживает множество языков программирования, добавить новые тоже не доставляет труда.

6. Toggle переключает ключевые слова: true/false, on/off, +/- и некоторые другие. Иногда бывает полезно.

7. Filetype completion позволяет выбрать тип файла из меню. Следует ввести :setf и выбрать тип файла с помощью клавиши Tab.

8. Pummode включает перемещение по меню дополнения (файлов, ключевых слов или omni completion) с помощью клавиш hjkl.

9. Repeat улучшает поведение стандартного повторения последнего действия, вызываемого клавишей ..

10. Surround устанавливает и удаляет «окружения» – скобки, html-теги и команды TeX. Например, ysw) заключит слово под курсором в круглые скобки, а ds) удалит их.

11. Enter-Indent правильно расставляет отступы в программном коде после нажатия Enter внутри скобок/тегов и т.д.

12. delinitMate – балансировка скобок, кавычек и т.д.

13. autoloadTemplate позволяет создавать шаблоны для разных типов файлов и автоматически загружает их при создании нового файла.

14. Timestamp обновляет время последнего редактирования файла. Настраивается просто, а в связке с предыдущим расширением и того проще, нужно лишь добавить в первые пять строк шаблона ключевые слова (настраиваются). Каждый короткий сценарий не помещать в систему контроля версий бессмысленно, а знать время создания и изменения бывает полезно.

15. Matchit расширяет навигацию между элементами с помощью %: можно передвигаться по парам if/else, begin/end и проч.

16. Easy-Motion добавляет множество новых способов передвижения и поиска, расширяющие стандартный поиск в строке с помощью fF и tT.

17. MRU сохраняет имена последних открытых файлов для быстрого доступа.

18. Yankring добавляет всё скопированное или удалённое в отдельный файл, имитируя менеджер буфера обмена. Мне это показалось в использовании более простым, чем сохранение данных в именованные регистры. Также удобно использовать один буфер обмена в разных «окнах» vim (редакторах, запущенных в разных сессиях).

19. quickhl выделяет цветом текст. Удобно использовать с другими плагинами, например, с look-up.

20. emmet-vim — плагин для zen-редактирования html. Например, html:5Ctrl + ,y — и будет создана основа html-документа с необходимыми тегами.

21. Neocomplcache — дополнение по файлам/буферам/ключевым словам и т.д. Использую этот плагин, т.к. пишу для него удобные сниппеты.

22. Lispcomplete — omnicompletion для Common Lisp.

23. haskellConceal — использование conceal-возможностей vim для «обёртывания» синтаксиса haskell математическими символами. Изменения только визуальные, подмены синтаксиса не происходит.

24. Rainbow Parenthesis. Как понятно из названия, это дополнение раскрашивает вложенные скобки. По умолчанию работает в файлах любого типа, но через ftdetect можно настроить для работы только, например, с искодным кодом на lisp/scheme (как у меня и сделано).

25. Error marker — навигация по ошибкам компиляции/интерпретации.

26. Tasklist отображает окно со ссылками на все «программистские» метки в файле, такие как TODO, FIXME, (DE)BUG и проч. Легко можно добавить свои шаблоны для поиска меток.

27. Number Marks — цифровые визуальные закладки. Удобнее стандартных возможностей vim (кроме, разве что, g;).

28. vCoolor. Дополнение позволяет вставлять коды цвета в разных форматах. Используется главным образом при редактировании таблиц стилей.

29. look-up — мой собственный плагин для интеграции sdcv (консольный клиент StarDict) в vim. Позволяет смотреть определения слов в словаре.

30. Figlet — ввод текста с помощью Figlet. Можно использовать :!figlet [...], но с помощью этого дополнения удобно выбирать нужный шрифт.

31. Improved AnsiEsc. Дополнение позволяет преобразовать escape-коды Ansi в видимую форму. Удобно использовать в конвеере с командами, имеющими цветной вывод.

32. Vimwiki. Этим плагином я пользуюсь ежедневно. Это полноценная вики с выводом в html и дневник (+календарь). Правда, я пользуюсь версией 1.1.1, т.к. в v2 сломали важные мне функции и привычное поведение.

33. Calendar интегрируется с VimWiki, позволяя вносить записи в дневник и показывать наличие записей в календаре.
